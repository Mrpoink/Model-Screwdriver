# Model Screwdriver: A Hyper-Aware Routing Engine for Rank-Based Transformer Updates 

Model Screwdriver is a novel, gradient-free framework designed for task-specific alignment. It utilizes a dual-headed routing and generation architecture to map low-rank concepts from a smaller "scout" model to a larger "target" model. By leveraging representation engineering and causal tracing, the system generates targeted weight edits that perform isolated manifold shifts without the computational overhead of standard Supervised Fine-Tuning (SFT).

---

## Dataset Structure

The dataset is synthetically generated via the `TaskHarvester` extraction pipeline. It is organized into shards to maintain manageable storage sizes and prevent VRAM exhaustion during training.

For evaluation results, the (`logs/`) directory contains all evaluation logs for all runs throughout `development`. (`logs/eval_logs_011-012`) are the latest logs we used, for which are the ones we utilized for the paper. You can also find a summarization of the final evaluation results, containing the information we used for the paper, in `final_results.json`

### Shard Composition
Each shard (`shard_xxxx.pt`) contains a list of dictionary records with the following keys:
* **`task_type`**: The specific textual domain (e.g., `imdb_sentiment`, `squad_qa`).
* **`A_small` / `B_small`**: Low-rank matrices ($r=6$) extracted from the scout model (BERT-Base).
* **`prompt_emb`**: Contextual embeddings for the task label generated by the scout model.
* **`A_large` / `B_large`**: Target weight matrices for the target model (BERT-Large).
* **`target_variance`**: A normalized probability curve of layer importance derived via **Causal Tracing**, serving as the ground-truth guide for the router.
